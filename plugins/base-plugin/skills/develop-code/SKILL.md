# Develop Code

## When to use

- Implementing a feature, bug, issue, hotfix

## Instructions

1. Read the asked
2. Read the code to understand how to implement it using standards in @rules and the best practices
3. Split in subtasks
4. Validate your plan with @pair-programming
5. Implement all subtask
6. Read the code again and compare to goal to check if it's ok
7. Call @code-review
8. Create a doc to explain what, where, when and why you did

## Important Notes
- Think simple
- Think in the long term (e.g. what I did is so essencial and connected that will create problem in the future? )
