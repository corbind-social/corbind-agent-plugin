# Understand Demand

## When to use

- Before create a plan
- Before to code

## Instructions

1. Validate if all required informations was passed or request for it
2. Validate if the DoR and DoD was passed or request for it
3. Validate if principal functional and technical test was passed or request for it
4. Check if you need to use @understand-demand or @understand-context
5. Create or update the spec file, plan file, issue with this informations
