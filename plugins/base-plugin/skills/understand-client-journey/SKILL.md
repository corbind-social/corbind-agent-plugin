# Understand Client Journey

## When to use

- Creating a plan
- Implementing a feature, bug, issue, hotfix

## Instructions

1. Use keywords to find important rules, docs, specs and other files about the journey you are/will implement
2. Check what files are important to read more
3. Compare the what you learn and what was asked to you 
4. Create a logical between both
5. Confirm with user if what you understand is correct
6. Maybe suggest some changes
7. Update plan or thinking about the implemantion
